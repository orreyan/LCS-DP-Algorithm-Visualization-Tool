📚 Algorithm Overview

Longest Common Subsequence (LCS) finds the longest subsequence common to two sequences.

Key Points:

  • Subsequence ≠ Substring (elements need not be contiguous)
  • Dynamic Programming solution: O(mn) time and space
  • Brute force would be O(2ⁿ) - exponential!
💡 Worked Example

Example: X = "ABC", Y = "AC"

Step 1: Compare 'A' with 'A' → Match → 1 + LCS("","") = 1

Step 2: Compare 'B' with 'C' → Mismatch → max(LCS("A",""), LCS("","C")) = 1

Result: LCS length = 2, LCS = "AC"

⚠️ Common Mistakes
❌ Mistake: Confusing subsequence with substring
✅ Fix: Remember subsequences don't need to be contiguous!
❌ Mistake: Forgetting to handle empty strings
✅ Fix: Always initialize first row/column with 0
📈 Complexity Analysis (CLO-1)

Time Complexity: O(m × n) where m = |X|, n = |Y|

Space Complexity: O(m × n) for DP table

Brute Force: O(2ⁿ) - checking all subsequences

Blue: DP O(mn) | Red: Brute Force O(2ⁿ)

🔍 Dynamic Programming (CLO-3)

Paradigm: Dynamic Programming

Recurrence Relation:

if X[i] == Y[j]:
    LCS[i][j] = 1 + LCS[i-1][j-1]
else:
    LCS[i][j] = max(LCS[i-1][j], LCS[i][j-1])

Overlapping Subproblems: Each cell depends on previously computed cells (top and left)

Optimal Substructure: LCS of prefixes leads to LCS of full strings

❓ Frequently Asked Questions

🔤 Basic Concepts

❓ What's the difference between subsequence and substring?
A substring must be consecutive characters (like "ABC" in "ABCD"). A subsequence can skip characters but maintains order (like "AC" in "ABCD").

Example: In "ABCBDAB", "BCB" is a subsequence but NOT a substring.

💡 Tip: Subsequence = Keep order, skip allowed | Substring = Consecutive only
❓ Is LCS the same as longest common substring?
NO! They are different:
• LCS: "ABC" vs "AC" → "AC" (length 2)
• Substring: "ABC" vs "AC" → "A" or "C" (length 1)

🎯 Algorithm Understanding

❓ How do I read the arrows in the DP table?
• ↖ (Diagonal): Characters MATCH! Value = top-left + 1
• ↑ (Up): Mismatch, take value from above
• ← (Left): Mismatch, take value from left
❓ Why can't we use a greedy algorithm?
Greedy FAILS because picking an early match might block a longer match later!
Example: "ABC" vs "ACB" → Greedy gives length 1, Optimal is 2.
